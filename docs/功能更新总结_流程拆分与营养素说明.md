# 功能更新总结：营养匹配流程拆分

## 更新时间
2026-03-11

## 更新内容

### 1. 流程拆分

将营养匹配功能拆分为两个独立流程，便于分步执行和调试：

#### 流程A：匹配 USDA ID
- **功能**：将 `out/ingredients.json` 中的食材匹配到 USDA SR Legacy 食材库的 id
- **输入**：`out/ingredients.json`
- **输出**：`out/matched_ingredients.json`
- **命令**：`python scripts/recipe_parser.py --match-usda-id`
- **特点**：需要人工操作验证匹配结果

#### 流程B：生成营养信息
- **功能**：根据 `matched_ingredients.json` 中的 USDA ID 获取营养信息
- **输入**：`out/matched_ingredients.json`
- **输出**：`out/nutritions.json`
- **命令**：`python scripts/recipe_parser.py --match-nutrition`
- **依赖**：需要先执行流程A

#### 组合执行
- **命令**：`python scripts/recipe_parser.py --match-usda-id --match-nutrition`
- **效果**：依次执行流程A和流程B

### 2. 核心代码修复

#### 修复单位提取问题
**问题**：USDA 数据中的单位字段名不正确

**修复**：
- 原代码：尝试从 `nutrient_data.unit` 获取单位
- 新代码：从 `nutrient.nutrient.unitName` 获取单位
- 影响：所有营养素单位现在都能正确提取

#### 修复单位匹配问题
**问题**：测量方法描述词导致单位匹配失败

**修复**：
- 实现 `normalize_unit()` 方法
- 标准化 Unicode 字符：`µ` (U+00B5) → `μ` (U+03BC)
- 移除测量方法描述词：
  - DFE（膳食叶酸当量）
  - NE（烟酸当量）
  - RAE（视黄醇活性当量）
  - α-TE（α-生育酚当量）
  - alpha-TE

**效果**：
- `folate`: 0% → 85.4% 成功率
- `niacin`: 0% → 88.2% 成功率
- `vitamin_a_rae`: 0% → 58.2% 成功率
- `vitamin_e`: 0% → 83.9% 成功率

#### 实现单位转换功能
**功能**：支持能量单位自动转换

**实现**：
- 添加 `convert_unit()` 方法
- 支持转换：`kJ ↔ kcal`
- 转换系数：1 kcal = 4.184 kJ

**使用场景**：
- USDA 数据中能量单位为 kJ，标准需要 kcal
- 系统自动转换并计算 nrp_pct
- 输出保留原始单位，添加转换备注

## 技术细节

### 关键代码修改

#### nutrition_generator.py
1. 新增 `normalize_unit()` 方法：标准化单位名称
2. 新增 `convert_unit()` 方法：支持单位转换
3. 修改 `calculate_nrv_values()` 方法：支持标准单位标准化
4. 拆分 `generate_nutrition_data()` 为：
   - `match_usda_ids()`: 流程A
   - `generate_nutrition_data_from_matched()`: 流程B
5. 修复 USDA 数据解析：正确提取 `nutrient.unitName` 字段

#### recipe_parser.py
1. 新增参数：`--match-usda-id`
2. 修改参数：`--match-nutritions` → `--match-nutrition`
3. 更新主逻辑：支持流程A和流程B的独立和组合执行
4. 更新执行模式判断：正确识别各种参数组合

### 设计原则遵循

1. **SOLID - 单一职责**：每个方法只做一件事
2. **DRY - 不重复**：避免重复的单位处理逻辑
3. **KISS - 保持简单**：清晰直观的单位转换规则
4. **YAGNI - 不过度设计**：只实现当前需要的功能

## 后续建议

### 可选改进

1. **添加更多单位转换**：如 mg ↔ g, μg ↔ mg
2. **支持更多营养素标准**：如欧盟标准
3. **添加营养素分类**：宏量/微量/维生素等

### 已知限制

1. **食材匹配依赖 AI**：需要人工验证匹配结果
2. **USDA 数据延迟**：不是实时更新
3. **部分营养素无标准**：标记为"无标准"但仍保留数据
4. **调料类高含量**：某些调料的营养素超过100%是正常的
