# 营养匹配流程使用说明

## 概述

营养匹配功能已拆分为两个独立流程，可以分别执行或组合执行。

## 流程说明

### 流程A：匹配 USDA ID

**功能：** 将 `out/ingredients.json` 中的食材匹配到 USDA SR Legacy 食材库的 id

**输入文件：**
- `out/ingredients.json` - 原始食材列表

**输出文件：**
- `out/matched_ingredients.json` - 匹配后的食材列表（包含 USDA ID）

**执行方式：**
```bash
python scripts/recipe_parser.py --match-usda-id
```

### 流程B：生成营养信息

**功能：** 根据 `matched_ingredients.json` 中的 USDA ID 获取详细的营养信息

**输入文件：**
- `out/matched_ingredients.json` - 已匹配的食材列表（必需）

**输出文件：**
- `out/nutritions.json` - 营养信息数据

**执行方式：**
```bash
python scripts/recipe_parser.py --match-nutrition
```

**注意：** 必须先执行流程A生成 `matched_ingredients.json`，否则流程B会报错。

## 组合执行

### 执行完整流程（流程A + 流程B）

```bash
python scripts/recipe_parser.py --match-usda-id --match-nutrition
```

或者：

```bash
python scripts/recipe_parser.py
```

（不添加任何营养匹配参数时，默认执行完整流程）

## 注意事项

1. **USDA 数据下载：** 无论执行哪个流程，都会自动下载、解压、解析 USDA SR Legacy 数据文件

2. **流程依赖：**
   - 流程B 依赖于流程A 的输出（`matched_ingredients.json`）
   - 如果单独执行流程B，请确保流程A 已经成功执行

3. **人工干预：**
   - 流程A（匹配 USDA ID）可能需要大量人工操作来验证和修正匹配结果
   - 建议先完成流程A，验证结果后再执行流程B

4. **临时文件清理：**
   - 执行完成后会自动清理临时目录中的 USDA 数据文件
   - `out/` 目录中的输出文件会被保留

## 常见问题

### Q: 如何只匹配 USDA ID？

```bash
python scripts/recipe_parser.py --match-usda-id
```

### Q: 如何只生成营养信息？

```bash
python scripts/recipe_parser.py --match-nutrition
```

### Q: 如何一次性完成整个流程？

```bash
# 方式1：使用默认行为
python scripts/recipe_parser.py

# 方式2：明确指定两个流程
python scripts/recipe_parser.py --match-usda-id --match-nutrition
```

### Q: 如果流程B报错 "matched_ingredients.json 不存在" 怎么办？

这表示还没有执行流程A。请先执行：

```bash
python scripts/recipe_parser.py --match-usda-id
```

### Q: 如何查看详细的执行日志？

```bash
python scripts/recipe_parser.py --verbose --match-usda-id
```

## 技术细节

### USDA 数据来源

- 主数据源：https://fdc.nal.usda.gov/fdc-datasets/FoodData_Central_sr_legacy_food_json_2018-04.zip
- 备用数据源：https://fdc.nal.usda.gov/fdc-datasets/FoodData_Central_sr_legacy_json_2021-10-28.zip

### 营养标准

- 中国 GB 28050-2011 NRV 参考值（优先使用）
- 美国 FDA Daily Value (DV) 参考值（作为补充）

### 匹配技能

使用 Claude Code 的 `match-ingredients` 技能进行语义匹配，确保匹配结果的准确性。
