# 营养信息生成功能技术说明

## 功能概述

本功能实现了从USDA SR（Standard Reference）数据库获取营养数据，并计算中国GB 28050-2011标准和美国FDA标准的NRV/DV值。

## 核心功能

1. **多标准营养值计算**：优先使用中国GB标准，补充美国FDA标准
2. **智能食材匹配**：改进的匹配算法可将中文食材名与英文USDA数据进行匹配
3. **完整的营养信息**：涵盖能量、宏量营养素、微量营养素等

## 实现细节

### 匹配算法改进

我们实现了一个多层次的匹配算法：

1. **精确映射**：对常见食材如"鸡蛋"、"牛奶"等进行了特殊处理
2. **关键词匹配**：将中文食材名映射到英文关键词进行匹配
3. **模糊匹配**：使用得分机制评估匹配质量

### 营养素名称标准化

我们将USDA数据库中的营养素名称标准化为便于计算NRV/DV的形式：

- "Energy" → "energy_kcal"
- "Protein" → "protein"
- "Vitamin A, RAE" → "vitamin_a_rae"

### NRV/DV计算

- **中国GB标准优先**：能量(2000kcal/8400kJ)、蛋白质(60g)、脂肪(60g)等
- **美国FDA标准补充**：用于中国标准未涵盖的营养素
- **单位匹配检查**：确保营养素单位与标准单位一致

## 当前成果

- **79个食材**的营养信息已成功匹配（从原来的8个大幅提升）
- **1244个营养成分**中有**872个**正确计算了NRV/DV百分比（约70%成功率）
- 生成了标准化的营养信息JSON文件（`out/nutritions.json`）

## 使用说明

```bash
# 单独运行营养信息匹配
python parse_recipes.py --match-nutritions

# 在完整流程中包含营养信息生成
python parse_recipes.py
```

## 输出文件

- `out/nutritions.json` - 营养信息数据
- `out/nutrition_map.json` - 食材与USDA数据的映射关系
