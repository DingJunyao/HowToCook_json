# parse-ingredients 技能

## 目标
给定一个食材列表，合并相同的食材为标准化格式的 JSON 数据，用于导入数据库。

## 输入
- 要导出 JSON 文件的路径
- 一个食材列表（可能来自多个菜谱解析结果，包含食材的名称、单位）

## 输出
- 在给定路径导出标准化的食材 JSON 数组，格式如下：
```json
[
  {
    "name": "食材名称",
    "aliases": ["别名1", "别名2", "别名3"],
    "category": "分类",
    "unit": "默认单位"
  },
  ...
]
```

## 分类体系
- grains: 谷物
- vegetables: 蔬菜
- fruits: 水果
- meat: 肉类
- seafood: 海鲜
- eggs: 蛋类
- dairy: 乳制品
- seasoning: 调味品
- oil: 油脂
- nuts: 坚果
- beverages: 饮品
- others: 其他

## 处理规则
- 不同菜谱里面对同种食材的叫法可能不一样，需要智能识别并合并
- 为每种食材建立别名列表，包含在不同菜谱中出现的不同叫法
- 根据食材特性正确分类
- 设置合理的默认单位（如：g、mL、个、条、根等）
- 智能识别同一种食材的不同表达方式（例如：土豆/马铃薯/洋芋 为同一食材）

## 注意事项
- 仅处理真正的食材，排除厨具、餐具、厨房用品
- 调料（盐、酱油、醋等）也属于食材范畴
- 合并时保留所有相关的别名
- 分类尽可能准确，不确定时归类为 "others"
